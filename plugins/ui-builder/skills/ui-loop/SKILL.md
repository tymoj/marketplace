# ui-loop

Autonomously build Angular pages one at a time from local design images.

Each cycle: reads baton → loads design image → builds Angular route component → registers lazy route → Playwright visual verification → updates SITE.md → writes next baton.

## Prerequisites

Your Angular project root must contain:

- **`SITE.md`** — app sitemap, roadmap, and router config (see `resources/site-template.md`)
- **`DESIGN.md`** — design system generated by the `design-md` skill
- **`designs/`** folder — local design images named after their routes (e.g. `designs/products.png`, `designs/checkout.jpg`)

## Baton File

Each cycle is driven by `next-prompt.md` in the project root. Format: see `resources/baton-schema.md`.

If `next-prompt.md` does not exist, read `SITE.md` and start from the first ⏳ Queued route in the roadmap.

## Workflow

### Step 1: Read Baton

Read `next-prompt.md`. Extract:
- `route` — Angular Router path
- `component` — PascalCase component class name
- `feature` — feature folder (kebab-case)
- `image` — path to the design image for this page

If no baton exists, read `SITE.md` → pick the first ⏳ Queued route → find its design image in `designs/[route].{png,jpg,jpeg,webp}`.

### Step 2: Load Context Files

Read these files to have full context:
- `SITE.md` — sitemap, roadmap, architecture, rules of engagement
- `DESIGN.md` — design system (Section 7 block is required for every component)
- `next-prompt.md` — any additional notes from the previous cycle

### Step 3: Read the Design Image

Use the **Read tool** to load the design image specified in the baton (`image` field).

Analyze the image using the image-to-angular skill workflow (Steps 2–4 of that skill): layout, colors, component inventory, form fields, interactive elements.

### Step 4: Validate Route is Not Duplicate

Check `SITE.md` sitemap — confirm the `route` does not already have status ✓ Complete or 🔄 In Progress. If it does, skip to Step 10 and pick the next queued route.

### Step 5: Build the Angular Route Component

Follow the full `image-to-angular` skill workflow (Steps 5–11):

1. Scaffold with `ng generate component [feature]/pages/[route] --standalone --style=scss --change-detection=OnPush --skip-tests`
2. Implement component with Angular 21 patterns (signals, OnPush, inject(), new control flow)
3. Apply design tokens from `DESIGN.md` Section 7
4. Implement forms with `FormBuilder.nonNullable` if the design contains form fields
5. Validate with `node scripts/validate.js` and `ng build`

### Step 6: Register Lazy Route

In `src/app/app.routes.ts`, add:

```typescript
{
  path: '[route]',
  loadComponent: () =>
    import('./[feature]/pages/[route]/[route].component')
      .then(m => m.[ComponentName]Component),
  title: '[Page Title from SITE.md]',
},
```

### Step 7: Update Navigation

Check `src/app/app.component.ts` for a `navLinks` array. If present, add the new route:

```typescript
navLinks = [
  // existing links...
  { path: '/[route]', label: '[Label]' },
];
```

### Step 8: Playwright Visual Verification

Start the dev server if not running:
```bash
ng serve &
```

Using Playwright MCP:
1. `playwright:browser_navigate` → `http://localhost:4200/[route]`
2. `playwright:browser_screenshot` → `screenshots/[route]-actual.png`
3. Compare screenshot with the design image read in Step 3
4. Fix layout/color/spacing deviations, rebuild, re-screenshot
5. Repeat until the component matches the design

### Step 9: Update SITE.md

Update the sitemap table:
- Change the route's status from ⏳ Queued → ✓ Complete
- Update the architecture tree (mark the component as `✓ Completed`)
- Update the Angular Router Config section with the new `loadComponent` entry
- Move the route from Roadmap "Current" to completed

### Step 10: Write Next Baton

Find the next ⏳ Queued route in `SITE.md`. Write `next-prompt.md`:

```yaml
---
route: [next-route]
component: [NextComponentName]
feature: [next-feature]
image: designs/[next-route].png
---
[One-line description of the next screen]

**DESIGN SYSTEM (REQUIRED):**
[Paste Section 7 from DESIGN.md verbatim]

**Angular Component Structure:**
- Layout: [infer from SITE.md architecture]
- Data: [mock data / HTTP service / resource()]
- [Any specific notes for this page]

**Page Structure:**
[Brief numbered list of main sections visible in designs/[next-route].png — to be confirmed when that image is read]
```

### Step 11: Confirm

Report to the user:
```
✓ Built: [ComponentName] → /[route]
  Image:  [image path]
  Files:  [list of created/modified files]

Next:   [NextComponentName] → /[next-route]
  Run "Run ui-loop to build the next page" to continue.
```

## Rules of Engagement (enforce throughout)

- Always read DESIGN.md Section 7 before writing any component
- All components: `standalone: true` + `ChangeDetectionStrategy.OnPush` + signals
- No `@Input()` / `@Output()` decorators — use `input()` / `output()` functions
- No hardcoded hex colors — use `--mat-*` CSS custom properties
- No `*ngFor` / `*ngIf` — use `@for (track)` / `@if`
- Routes must be lazy-loaded via `loadComponent()`
- Never duplicate a page — always check SITE.md sitemap first
- File naming: kebab-case for all files and folders
