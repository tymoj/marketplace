# ui-builder

Convert local design images into production Angular 21 components with Angular Material.

Drop in a PNG, JPG, Figma export, or screenshot — get back a pixel-close, fully working Angular 21 standalone component with reactive forms, correct validators, and Angular Material M3 styling. No Stitch MCP. No external design tools required.

## Quick Start

**1. Install the plugin**

```bash
claude plugin install ui-builder@tymoj/marketplace
```

**2. Add MCPs to `.claude/settings.json`** in your project

```json
{
  "mcpServers": {
    "angular": {
      "command": "npx",
      "args": ["-y", "@angular/cli", "mcp"]
    },
    "playwright": {
      "command": "npx",
      "args": ["-y", "@playwright/mcp@latest"]
    }
  }
}
```

**3. Restart Claude Code** so the MCPs are picked up.

**4. Convert a design image**

```
Use image-to-angular to convert designs/checkout.png into a CheckoutFormComponent
```

Or build your full app page by page:

```
Run ui-loop to build the next page in my Angular app's roadmap
```

## How It Works

Claude reads design images natively using the **Read tool** — the same way it reads any file. No browser, no external service. It analyzes colors, layout, form fields, typography, and component types directly from the image, then generates Angular 21 code that matches.

For forms, it detects every field, infers validators from context (required indicators, email fields, password rules), and generates a `FormBuilder.nonNullable` reactive form with proper `<mat-error>` messages.

Playwright MCP takes a screenshot of the rendered component and compares it to the original design, fixing deviations until they match.

## Skills

| Skill | What it does | Trigger |
|---|---|---|
| `design-md` | Read design images → extract tokens → write `DESIGN.md` | `Use design-md on designs/home.png` |
| `image-to-angular` | Read image → build Angular 21 component | `Use image-to-angular to convert designs/login.png into LoginComponent` |
| `enhance-prompt` | Turn a rough description into a full Angular component spec | `Use enhance-prompt to refine this component description` |
| `angular-material` | Set up Angular Material theming in an existing project | `Use angular-material to set up theming` |
| `ui-loop` | Autonomous loop — build every page from your `designs/` folder | `Run ui-loop to build the next page` |

## ui-loop (autonomous page building)

`ui-loop` builds Angular pages one at a time from local design images. It needs:

- **`SITE.md`** — app sitemap, roadmap, router config (use `skills/ui-loop/resources/site-template.md`)
- **`DESIGN.md`** — design system generated by `design-md`
- **`designs/`** folder — images named after their routes (`products.png`, `checkout.jpg`, etc.)

Each cycle: reads `next-prompt.md` baton → reads design image → builds Angular route component → registers lazy route → Playwright screenshot verification → updates `SITE.md` → writes next baton.

## Using with conductor

When conductor runs Phase 3 for Angular frontend tasks, `angular-expert` automatically uses `image-to-angular`, `angular-material`, and `design-md` skills. Include design image paths in your task description:

```
/conductor:dev-plan Build the checkout page based on designs/checkout.png
```

Conductor will plan the component architecture, and the frontend teammate will use ui-builder skills to implement it from the image.

See `plugins/conductor/agents/angular-expert.md` for the skill list configuration.

## Angular 21 Patterns

All generated components use:

- `standalone: true` — no NgModules
- `ChangeDetectionStrategy.OnPush` — everywhere
- `input()` / `output()` — signal-based I/O, never `@Input`/`@Output`
- `signal()`, `computed()`, `effect()` — reactive state
- `resource()` / `rxResource()` — async data fetching
- `inject()` — function-based DI, never constructor injection
- `@for (track)`, `@if`, `@switch`, `@let` — new control flow
- `FormBuilder.nonNullable` — reactive forms, never template-driven for complex forms
- `--mat-*` CSS custom properties — never hardcoded hex colors
