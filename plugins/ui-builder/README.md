# ui-builder

Stitch Skills adapted for **Angular 21**, **Angular CLI MCP**, and **Playwright MCP**.

Converts Google Stitch designs into production Angular 21 route components, design systems, and full app flows.

Original: https://github.com/google-labs-code/ui-builder | License: Apache-2.0

## Quick Start

**1. Install the plugin**

```bash
claude plugin install ui-builder@tymoj/marketplace
```

**2. Add MCPs to `.claude/settings.json`** in your project

```json
{
  "mcpServers": {
    "angular": {
      "command": "npx",
      "args": ["-y", "@angular/cli", "mcp"]
    },
    "playwright": {
      "command": "npx",
      "args": ["-y", "@playwright/mcp@latest"]
    }
  }
}
```

**3. Restart Claude Code** so the MCPs are picked up.

**4. Start using a skill** — e.g. to build your Angular app page by page:

```
Run stitch-loop to build the next page in my Angular app's roadmap
```

## Using with conductor

If you have the `conductor` plugin installed, add the Stitch skills to `angular-expert` so they are available during Phase 3 implementation:

**`plugins/conductor/agents/angular-expert.md`** — add to the `skills:` list:

```yaml
skills:
  - rest-api-design
  - frontend-design
  - testing-strategy
  - design-md
  - angular-components
  - angular-material
```

Then reinstall both plugins:

```bash
claude plugin install conductor@tymoj/marketplace
claude plugin install ui-builder@tymoj/marketplace
```

Or locally:

```bash
bash plugins/conductor/install.sh
bash plugins/ui-builder/install.sh
```

After this, when conductor routes to `angular-expert` in Phase 3, it will apply Stitch design tokens, Angular Material theming, and Angular 21 component patterns automatically.

## Stitch Loop (autonomous page building)

`stitch-loop` iteratively builds Angular pages one at a time. It requires two files in your Angular project root:

- **`SITE.md`** — app sitemap, roadmap, and router config. Use `skills/stitch-loop/resources/site-template.md` as a starting point.
- **`DESIGN.md`** — design system generated by the `design-md` skill.

Each cycle: reads baton → generates Stitch screen → converts to Angular route component → registers lazy route → verifies with Playwright → updates `SITE.md` sitemap → writes `next-prompt.md` baton for the next cycle.

## Angular 21 Patterns

All generated components use:

- `standalone: true` — no NgModules
- `ChangeDetectionStrategy.OnPush` — everywhere
- `input()` / `output()` — signal-based I/O (not `@Input`/`@Output`)
- `signal()`, `computed()`, `effect()` — reactive state
- `resource()` / `rxResource()` — async data fetching
- `inject()` — function-based DI (not constructor injection)
- `@for`, `@if`, `@switch`, `@let` — new control flow
- `provideZonelessChangeDetection()` — zoneless by default
